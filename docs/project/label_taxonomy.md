# GovAI Label Taxonomy

This document describes the label categories used for GovAI issue triage and roadmap tracking.

## Core labels

- bug: Unexpected behavior, correctness issue, or regression.
- enhancement: New feature or improvement.
- documentation: Documentation updates.
- research: Research-oriented work or experiments.
- security: Security-related changes or vulnerabilities.

## Contributor labels

- good first issue: Suitable for new contributors.
- help wanted: Community contributions welcome.
- discussion: Open design or architecture discussion.

## Priority labels

- priority-high: High priority work.
- priority-medium: Medium priority work.
- priority-low: Low priority work.

## Architecture labels

- audit-ledger: Ledger integrity and audit event storage.
- evidence-bundles: Evidence packs, replay, verification, and artifact continuity.
- runtime-enforcement: Runtime blocking and enforcement logic.
- policy-engine: Declarative policy engine and evaluation.
- governance-runtime: Runtime governance orchestration and enforcement.
- tenant-isolation: Tenant separation and isolation guarantees.
- RBAC: Role-based access control and authorization.

## Developer experience labels

- developer-experience: Developer workflow and ergonomics.
- onboarding: Contributor onboarding improvements.
- examples: Example projects and demos.
- cli: CLI commands and workflows.
- ci: Continuous integration workflows.

## Integration labels

- integrations: External integrations.
- OpenTelemetry: OpenTelemetry tracing integration.
- MLflow: MLflow integration work.
- LangChain: LangChain integration work.
- github-actions: GitHub Actions workflows.

## Compliance labels

- ai-act: EU AI Act related functionality.
- governance-as-code: Governance definitions as code.
- traceability: Decision traceability and provenance.
- auditability: Auditability and evidence guarantees.
- cryptographic-verification: Cryptographic signing and integrity verification.
